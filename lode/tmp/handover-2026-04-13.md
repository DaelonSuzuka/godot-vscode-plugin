## Key Technical Findings

### 1. @ Token Issue (SOLVED)
The `@` character for annotations must be the **first** alternative in statement rules:
```lezer
statement {
  "@" identifier    // MUST BE FIRST
  | classNameKw identifier
  | ...
}
```

### 2. Keywords Need @specialize
```lezer
varKw { @specialize[@name=var]<identifier, "var"> }
funcKw { @specialize[@name=func]<identifier, "func"> }
```

### 3. Binary Operators Need @precedence
```lezer
@precedence { expr @left }

expr {
  expr !expr ("+" | "-") expr
  | expr !expr ("*" | "/") expr
  | identifier
  | Number
}
```

## Big Missing Piece: Function Bodies

The grammar doesn't handle indented code blocks yet:
```gdscript
func foo():
    var x = 5      # <-- can't parse this
    print(x)
```

This requires implementing Lezer's indentation tracking with external tokenizers.
